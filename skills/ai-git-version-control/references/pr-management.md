## Agent PR Template

Every agent-generated PR MUST include these sections:

```markdown
## Task Description
<Original task description> (ID: <task-id>)

## What Changed
<Summary of core changes>

## Key Design Decisions
<Rationale for important decisions>

## Alternatives Considered
<Other approaches evaluated and why they were rejected>

## Test Coverage
<What was tested and how>

## Known Limitations / Follow-up Tasks
<What's not covered and what needs follow-up>

## Review Guidance
<Where reviewers should focus attention>

---
*This PR was generated by AI agent following Agent-Aware Git best practices*
```

**Script:** `scripts/generate-agent-pr-template.sh <task_id> <task_description> [changes] [decisions] [alternatives] [tests] [limitations] [review_guidance]`

## PR Creation Workflow

```
1. Clean up commit history (squash [WIP] commits, ensure trailers)
2. Push branch to remote
3. Generate PR template using script
4. Create PR via gh cli:
   gh pr create --base main --head agent/<task-id>-<desc> \
     --title "<type>(<scope>): <summary>" \
     --body-file .github/pull_request_template/agent.md
5. Wait for CI checks
6. Request review
```

### Rules
- Never merge your own PRs
- All CI checks must pass before requesting review
- PR title must follow Conventional Commits format
- Keep PRs small and focused (one logical change per PR)

## Stacked PRs

For large features that need to be broken into sequential, dependent PRs.

### When to Use Stacked PRs
- Feature requires multiple logical changes that must merge in order
- Each PR builds on the previous one
- Reviewers benefit from reviewing incrementally

### Workflow
```
main ─── PR #1 (base) ─── PR #2 (depends on #1) ─── PR #3 (depends on #2)
```

### Creating Stacked PRs

1. Create a file with PR titles (one per line):
```
feat(auth): add refresh token model
feat(auth): implement token rotation endpoint
feat(auth): integrate rotation into login flow
```

2. Run the script:
```bash
scripts/create-stacked-prs.sh main pr-titles.txt
```

**Prerequisites:** `gh` CLI installed and authenticated, `gh-stack` (optional, for sync)

### Stacked PR Management
- Use `gh-stack` to keep PRs in sync when rebasing
- When a lower PR changes, all PRs above must be updated
- Merge from bottom to top

## Review Guidance

When filling the "Review Guidance" section in the PR template:

1. **Focus areas**: Which files/functions need the most attention
2. **Risk assessment**: What could break and why
3. **Testing suggestions**: How to manually verify the changes
4. **Known concerns**: Areas where the agent is less confident

## Commit History Cleanup Before PR

Before opening a PR, clean up the commit history:

1. **Squash [WIP] checkpoint commits** into their corresponding semantic commits
2. **Reword** any unclear commit messages to follow Conventional Commits
3. **Ensure all commits** have Agent-Task/Model/Decision trailers
4. **Verify atomic commits**: each commit should be independently buildable

**Script:** `scripts/agent-rebase-history.sh [main_branch] [commit_count]`

### Interactive Rebase Commands
```bash
# Start interactive rebase
git rebase -i main HEAD~<N>

# In the editor:
# pick   = keep this commit
# squash = merge into previous commit
# reword = change commit message
# fixup  = squash, discard this commit's message
# drop   = remove this commit entirely
```
