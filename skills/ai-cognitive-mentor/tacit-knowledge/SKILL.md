# 默会知识（Tacit Knowledge / Polanyi式）

## 作用

挖掘专家知道但无法完全表达的隐性经验、直觉判断和情境智慧。

## 核心概念

**默会知识（Tacit Knowledge）**：由迈克尔·波兰尼（Michael Polanyi）提出，指"我们知道的比我们能说出的多"。这类知识难以用语言文字完全表达，只能通过实践、师徒制、情境体验来传递。

**显性知识（Explicit Knowledge）**：可以用文字、数字、公式、手册等形式系统表达的知识。

### 两者对比

| 维度 | 显性知识 | 默会知识 |
|------|---------|---------|
| 表达形式 | 文字、公式、手册 | 经验、直觉、诀窍 |
| 获取方式 | 读书、听课、搜索 | 实践、师徒、沉浸式体验 |
| 转移难度 | 容易 | 困难 |
| 存储位置 | 文档、数据库 | 人的头脑、身体、组织文化 |
| 典型例子 | 数学定理、操作手册 | 骑单车的平衡感、识人眼光 |

## 默会知识的表现形式

### 1. 直觉判断（Intuition）
- 专家"一眼就看出问题在哪"，但说不清为什么
- 基于大量经验积累的模式识别

### 2. 情境智慧（Situational Judgment）
- 知道"什么时候该怎么做"
- 能根据具体情况灵活调整，而不是生搬硬套规则

### 3. 不成文规则（Unwritten Rules）
- 行业潜规则、组织惯例
- "大家都这么做，但没人写在手册里"

### 4. 质量感知（Quality Perception）
- 能分辨好坏、优劣、正常/异常
- 说不清标准，但"看到就知道"

### 5. 关系网络（Relational Know-how）
- 知道找谁、怎么沟通、如何协调
- 人际关系中的默契和分寸

## 提取默会知识的方法

### 方法 1：关键事件回顾（Critical Incident Technique）

让专家回忆印象深刻的具体事件，从细节中挖掘默会知识。

**提问框架**：
- "回忆一个你处理得特别好的案例，当时具体发生了什么？"
- "你最先注意到了什么？是什么让你警觉？"
- "你当时是怎么判断的？脑子里想了什么？"
- "如果换一个新手来，他最可能在哪里出错？"

### 方法 2：新手-专家对比

对比新手和专家在同一问题上的不同做法，找出差异点。

**分析框架**：
- 专家注意到了什么新手没注意到的？
- 专家跳过了什么新手会纠结的？
- 专家的优先级排序和新手有什么不同？
- 专家的判断标准是什么？

### 方法 3：失败案例反推

从"什么不能做"反推"为什么不能做"，挖掘隐性约束。

**提问框架**：
- "新手最容易犯什么错？"
- "哪些做法看起来对，但实际上行不通？"
- "你踩过的最大的坑是什么？你从中学到了什么？"
- "有没有什么规则，违反了虽然不会立刻出事，但长期一定会出问题？"

### 方法 4：边界条件探测

询问规则的适用边界，发现隐含的适用条件。

**提问框架**：
- "这个方法在什么情况下特别好用？"
- "在什么情况下这个方法不适用？为什么？"
- "你怎么判断该用方法A还是方法B？"
- "有没有看似适用但其实不适用的情况？"

### 方法 5："如果你是我"提问

让专家代入你的角色，给出具体行动建议。

**提问框架**：
- "如果你是我，面对这个情况，你第一步会做什么？"
- "你会先看什么信息？先找谁？"
- "你判断这件事的优先级标准是什么？"
- "如果只能做一件事，你会做什么？为什么？"

## 适用场景

| 场景 | 为什么适合 |
|------|-----------|
| 实操性咨询 | 很多实操知识只可意会不可言传 |
| 技能传承 | 老师傅带徒弟，需要把隐性经验传递下去 |
| 情境决策 | 复杂情境下的判断，无法完全靠规则手册 |
| 行业潜规则 | 了解行业内不成文的规矩和惯例 |
| 组织知识管理 | 把老员工脑子里的知识挖出来，避免人走知识走 |

## 常见陷阱

1. **过度理性化**：试图把一切都变成显性知识，忽略默会知识的不可言传性
2. **只问"怎么做"，不问"怎么判断"**：操作步骤容易学，判断标准才是核心
3. **不验证**：专家说的不一定都对，需要结合实际情况验证
4. **忽视情境**：默会知识高度依赖具体情境，不能生搬硬套
5. **以为听过就会了**：默会知识需要实践内化，不是听一遍就能掌握

## 与其他工具的配合

- **mental-models**：心智模型是显性化的默会知识，帮助把经验变成可传递的框架
- **first-principles-thinking**：第一性原理从基础推导，默会知识从经验归纳，两者互补
- **structured-output**：用结构化输出整理挖掘到的默会知识，便于传递和积累
