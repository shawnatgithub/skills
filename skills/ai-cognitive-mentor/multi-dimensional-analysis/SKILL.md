# 多维度/跨学科分析（Multi-dimensional Analysis）

## 作用

从多个维度（技术、经济、心理、社会、伦理、历史等）进行多视角分析，避免单一视角的盲区。

## 核心原则

1. **每个维度都有盲区**：单一维度的分析必然有盲点
2. **维度之间相互影响**：一个维度的变化会牵动其他维度
3. **不是维度越多越好**：抓住核心的 3-7 个维度，避免维度泛滥
4. **最终要整合**：多维度不是目的，目的是形成对问题的整体理解

## 常用分析维度

### 1. 技术维度（Technical）

**关注点**：可行性、效率、可扩展性、可靠性

**关键问题**：
- 技术上是否可行？有没有现成方案？
- 性能、效率、质量如何？
- 技术风险有哪些？
- 未来技术演进方向是什么？

### 2. 经济维度（Economic / Financial）

**关注点**：成本、收益、投资回报、资源分配

**关键问题**：
- 成本多少？收益多少？ROI 如何？
- 谁付费？谁受益？成本如何分摊？
- 短期成本 vs 长期收益如何平衡？
- 有哪些隐性成本？

### 3. 心理维度（Psychological）

**关注点**：动机、认知偏差、行为模式、情绪

**关键问题**：
- 人的动机是什么？什么驱动行为？
- 可能存在哪些认知偏差？
- 情绪因素如何影响决策？
- 接受度如何？会不会有抵触？

### 4. 社会维度（Social / Cultural）

**关注点**：社会关系、群体行为、文化规范、公平性

**关键问题**：
- 对不同群体的影响有何不同？
- 社会规范、文化习俗如何作用？
- 公平性如何？会不会加剧不平等？
- 社会网络效应如何？

### 5. 伦理维度（Ethical / Moral）

**关注点**：对错、正义、责任、权利

**关键问题**：
- 涉及哪些伦理争议？
- 谁承担风险？谁享有收益？
- 弱势群体是否得到保护？
- 长期来看，这会让世界变得更好还是更坏？

### 6. 历史维度（Historical）

**关注点**：演变路径、历史教训、长期趋势

**关键问题**：
- 历史上类似的情况是怎样的？
- 我们是怎么走到今天这一步的？
- 有什么历史经验教训可以借鉴？
- 这是周期性的还是结构性的？

### 7. 法律/规制维度（Legal / Regulatory）

**关注点**：合规性、法律风险、监管环境

**关键问题**：
- 相关法律法规有哪些？
- 合规成本多少？法律风险如何？
- 监管趋势是趋严还是放松？
- 不同司法管辖区有何差异？

### 8. 环境维度（Environmental）

**关注点**：环境影响、可持续性、资源约束

**关键问题**：
- 对环境的影响是什么？
- 资源消耗是否可持续？
- 碳足迹如何？
- 有没有环境风险？

### 9. 政治维度（Political）

**关注点**：权力关系、利益集团、政策博弈

**关键问题**：
- 涉及哪些利益相关方？他们的权力如何？
- 政治可行性如何？
- 政策制定过程是怎样的？
- 有没有政治风险？

## 执行步骤

### 步骤 1：选择核心维度

根据问题性质，选择 3-7 个最相关的维度。不是每个问题都需要所有维度。

**维度选择原则**：
- 与问题直接相关
- 每个维度提供独特视角，不重叠
- 覆盖主要利益相关方的关注点

### 步骤 2：逐维度分析

对每个选定的维度，深入分析该维度下的问题表现、关键因素、影响机制。

### 步骤 3：维度交叉分析

分析不同维度之间的相互作用和冲突。

**关键问题**：
- 哪些维度之间有协同效应？
- 哪些维度之间有矛盾冲突？
- 一个维度的变化会如何影响其他维度？
- 有没有"一举多得"的杠杆点？

### 步骤 4：整合与权衡

综合各维度的分析，形成整体判断，明确取舍和权衡。

**输出格式**：
```
【多维度分析】
主题：___

【选定维度】___个

## 1. ___维度
- 核心发现：___
- 关键因素：___
- 风险/机会：___

## 2. ___维度
- 核心发现：___
- 关键因素：___
- 风险/机会：___

## 3. ___维度
- 核心发现：___
- 关键因素：___
- 风险/机会：___

[...更多维度...]

## 维度交叉分析
- 协同效应：___
- 冲突矛盾：___
- 关键权衡：___
- 杠杆点：___

## 综合判断
- 整体评估：___
- 主要优势：___

```

## 常用分析框架组合

### 商业战略分析
- 技术 + 经济 + 市场 + 竞争 + 组织

### 公共政策评估
- 经济 + 社会 + 伦理 + 法律 + 政治

### 产品设计
- 用户（心理） + 技术 + 商业 + 社会 + 伦理

### 个人重大决策
- 经济 + 心理 + 健康 + 关系 + 成长

## 适用场景

| 场景 | 为什么适合 |
|------|-----------|
| 战略决策 | 战略问题涉及多方面因素，单一视角必然偏颇 |
| 社会议题 | 社会问题复杂，需要从多个角度理解 |
| 产品定位 | 产品要在技术、商业、用户、社会等多重约束中找平衡 |
| 政策评估 | 政策影响广泛，需要全面评估利弊 |
| 复杂问题诊断 | 多维度分析有助于找到问题的真正根源 |

## 常见陷阱

1. **维度泛滥**：列了十几个维度，每个都浅尝辄止，没有深度
2. **平均用力**：每个维度花同样多的精力，没有聚焦核心维度
3. **只罗列，不整合**：各维度分析完就结束了，没有形成整体判断
4. **忽视维度间的相互作用**：把维度当成分离的，忽略了联动效应
5. **选择性维度**：只选支持自己观点的维度，回避不利的维度（确认偏误）

## 与其他工具的配合

- **systems-thinking**：多维度分析提供视角，系统思维分析各维度间的互动关系
- **second-order-effects**：多维度分析空间上的广度，第二序效应分析时间上的深度
- **mental-models**：每个维度都可以调用相应的心智模型深化分析
