# 第一性原理（First Principles Thinking）

## 作用

把问题分解为基本的真理和公理，避免依赖类比、惯常做法或现有解决方案。从头重建。

## 核心原则

1. **解构而非类比**：不要用"过去怎么做"或"别人怎么做"来推理
2. **逐层追问"为什么"**：连续追问直到触及不可再分的基本事实
3. **区分"已知为真"与"假设为真"**：严格区分已验证的事实和人们信以为真的观念
4. **从基础向上构建**：确认基本真理后，再逐步推导解决方案

## 执行步骤

### 步骤 1：识别并写下当前的假设

列出你对这个问题的所有现有假设和信念。

**输出格式**：
```
【当前假设清单】
1. 假设A：___（来源：___）
2. 假设B：___（来源：___）
3. 假设C：___（来源：___）
```

### 步骤 2：将问题分解为基本真理

将问题逐层拆解，直到达到不可再分的、可以被验证为真的基本要素。

**追问方法**：
- "这是真的吗？我怎么知道它是真的？"
- "如果去掉这个前提，问题还成立吗？"
- "这个要素由什么构成？能否再拆？"

**输出格式**：
```
【基本真理清单】
1. 真理A：___（验证方式：___）
2. 真理B：___（验证方式：___）
3. 真理C：___（验证方式：___）

【被推翻的假设】
- 假设X：___（原因：___）
```

### 步骤 3：从基本真理出发重新构建解决方案

不依赖任何被推翻的假设，仅从验证后的基本真理出发，重新推导解决方案。

**构建方法**：
- 从最核心的 1-2 条基本真理开始
- 逐步加入其他真理作为约束条件
- 每一步都问："如果这是真的，那么什么必然成立？"

**输出格式**：
```
【从第一性原理推导的解决方案】
核心出发点：___（基本真理）
→ 推论1：___
→ 推论2：___
→ 推论3：___
→ 最终方案：___

【与常规方案的对比】
- 常规方案：___
- 第一性原理方案：___
- 关键差异：___
```

## 适用场景

| 场景 | 为什么适合 |
|------|-----------|
| 根本性创新 | 打破现有范式，从零开始重新定义 |
| 突破瓶颈 | 当增量改进失效时，回到基础重新思考 |
| 复杂归因 | 剥离层层表象，找到问题的真正根源 |
| 跨界迁移 | 将一个领域的基本真理应用到另一个领域 |
| 验证直觉 | 用基本真理检验"感觉不对"的方案 |

## 常见陷阱

1. **停在半中间**：拆解不够深，把"行业常识"当成了基本真理
2. **事实与观点混淆**：把大多数人相信的观点当作了事实
3. **只拆不建**：做了大量拆解，但没有从基础重新构建解决方案
4. **忽视现实约束**：基本真理是对的，但忽略了时间、资源等硬约束

## 与其他工具的配合

- **socratic-inversion**：先用苏格拉底式质疑挑战假设，再用第一性原理重建
- **systems-thinking**：第一性原理拆解要素，系统思维分析要素间的关系
- **mental-models**：第一性原理提供基础，心智模型提供构建框架
