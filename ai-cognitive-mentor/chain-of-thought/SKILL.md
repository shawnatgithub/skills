# 链式思考 + 显性推理（Chain of Thought）

## 作用

把推理过程显性化，减少跳步错误。

## 核心原则

1. **每一步都写出来**：不要省略中间步骤，哪怕看起来显而易见
2. **每步只推进一小步**：确保每一步之间的逻辑跨度足够小
3. **标明每步的依据**：每一步推导都要有明确的前提或规则支撑
4. **可以回退**：如果发现某一步错了，可以回退到上一个正确的步骤

## 基础模式：逐步推理

### 执行步骤

1. 明确已知条件和待证结论
2. 从已知出发，一步一步推导
3. 每一步都说明"从什么，根据什么，得到什么"
4. 最终到达结论

**输出格式**：
```
【链式推理】
已知条件：
1. ___
2. ___

待证/待求：___

推理过程：
步骤1：由___，根据___，可得___
步骤2：由步骤1和___，根据___，可得___
步骤3：由步骤2和___，根据___，可得___
...
结论：___
```

## 进阶变体 1：思维树（Tree of Thoughts）

当问题有多条推理路径时，探索多个分支并评估。

### 适用场景
- 有多种可能的解决方案需要比较
- 推理过程中会遇到分叉点
- 需要回溯和尝试不同路径

### 执行步骤

1. 生成多个初始思路（2-5个）
2. 对每个思路进行几步推演
3. 评估每个思路的可行性和前景
4. 放弃差的路径，深入好的路径
5. 最终选择最优路径得出结论

**输出格式**：
```
【思维树推理】
问题：___

初始分支：
分支A：___
分支B：___
分支C：___

各分支推演与评估：
分支A：
- 推演：___
- 评估：可行性___，前景___（保留/放弃）

分支B：
- 推演：___
- 评估：可行性___，前景___（保留/放弃）

分支C：
- 推演：___
- 评估：可行性___，前景___（保留/放弃）

最终选择：分支___
深入推演：
步骤1：___
步骤2：___
...
结论：___
```

## 进阶变体 2：自洽性（Self-Consistency）

生成多条独立推理路径，选择结果最一致的那个。

### 适用场景
- 问题有唯一正确答案，但推理路径可能出错
- 需要降低偶然推理错误的概率
- 重要决策需要多重验证

### 执行步骤

1. 独立地生成 3-5 条不同的推理路径
2. 每条路径得出自己的结论
3. 比较各条路径的结论
4. 选择出现次数最多、最自洽的结论

**输出格式**：
```
【自洽性验证】
问题：___

路径1：
- 推理：___
- 结论：___

路径2：
- 推理：___
- 结论：___

路径3：
- 推理：___
- 结论：___

一致性分析：
- 结论一致的路径：___条
- 存在分歧的点：___
- 最终结论：___
```

## 适用场景

| 场景 | 为什么适合 |
|------|-----------|
| 精确计算/推导 | 一步步算，避免算术错误 |
| 复杂决策 | 把决策过程显性化，便于审视和检验 |
| 代码调试 | 逐步跟踪执行流程，定位 bug |
| 逻辑谬误检查 | 每步都检验，容易发现逻辑漏洞 |
| 教学/解释 | 让他人能跟随你的思考过程 |

## 常见陷阱

1. **跳步**：跳过关键步骤，导致错误难以发现
2. **循环论证**：用结论本身作为前提
3. **隐含前提不说明**：把没有明确说明的假设当作已知
4. **只列步骤，不讲依据**：每一步都需要说明"为什么可以这么推"
5. **路径依赖**：一条路走到黑，不考虑其他可能性

## 注意事项

⚠️ **对创意发散类问题（头脑风暴、品牌命名）慎用**，链式思考会抑制跳跃性思维，反而限制创意的产生。

## 与其他工具的配合

- **socratic-inversion**：用苏格拉底式质疑检验每一步推理的合理性
- **systems-thinking**：链式思考走通单一路径，系统思维看整体结构
- **structured-output**：用结构化输出整理最终结论
