# 结构化输出（Structured Output）

## 作用

将回答分为：核心结论 + 详细推理链 + 潜在风险/反例 + 更好替代方案 + 行动步骤。确保输出清晰、有条理、可执行。

## 核心原则

1. **结论先行**：先给答案，再解释。不要让读者自己找重点
2. **层次分明**：用标题、列表、表格等方式组织内容
3. **可扫读**：读者扫一眼就能抓住核心，感兴趣再看细节
4. **可执行**：不仅有分析，还要有具体的行动建议
5. **诚实面对不确定性**：知道就是知道，不确定就说不确定

## 标准输出结构

### 五段式标准结构

```
【核心结论】
一句话说清楚结论。

【详细分析/推理链】
为什么得出这个结论？推理过程是怎样的？
（分点展开，每点有依据）

【潜在风险与反例】
- 风险1：___（应对：___）
- 风险2：___（应对：___）
- 反例：在什么情况下这个结论不成立？

【替代方案】
如果这个方案不行，还有什么选择？
- 方案A：___（优缺点：___）
- 方案B：___（优缺点：___）

【行动步骤】
1. 第一步：___（负责人/时间：___）
2. 第二步：___（负责人/时间：___）
3. 第三步：___（负责人/时间：___）
```

【执行要点】
1. ___

## 结构化技巧

### 1. 用数字说话

- 模糊："效果很好"
- 清晰："效率提升了 40%，从每天 100 单提升到 140 单"

### 2. 用对比凸显差异

- 平铺直叙："新方案更快"
- 对比呈现："原方案需要 3 天，新方案只需要 1 天，节省 67% 的时间"

### 3. 用分类降低认知负荷

把相关的东西归为一类，每类不超过 7 项（人脑工作记忆的限制）。

**常见分类方式**：
- 按时间：短期 / 中期 / 长期
- 按重要性：核心 / 次要 / 可选
- 按维度：技术 / 商业 / 用户
- 按逻辑：原因 / 过程 / 结果

### 4. 用表格整理对比

需要比较多个选项时，表格最清晰。

```
| 维度 | 方案A | 方案B | 方案C |
|------|-------|-------|-------|
| 成本 | 高 | 中 | 低 |
| 速度 | 快 | 中 | 慢 |
| 风险 | 低 | 中 | 高 |
| ... | ... | ... | ... |
```

### 5. 用优先级排序

不是所有事情都一样重要。明确标出优先级。

**优先级标记方式**：
- P0 / P1 / P2 / P3
- 🔴 必须 / 🟡 应该 / 🟢 可以
- 高 / 中 / 低

### 6. 诚实地说"不知道"

不知道就说不知道，不要瞎编。

**好的说法**：
- "我不确定，但根据现有信息，我倾向于认为___"
- "这方面我需要更多数据才能下结论"
- "有两种可能，目前的证据还不足以判断哪一种对"

## 适用场景

| 场景 | 为什么适合 |
|------|-----------|
| 最终交付物 | 结构化的输出更容易理解和使用 |
| 长篇分析 | 结构帮助读者导航，找到自己关心的部分 |
| 行动指南 | 明确的步骤让建议可执行 |
| 决策汇报 | 结论先行，便于决策者快速抓住重点 |
| 知识传递 | 结构化的内容更容易记忆和复用 |

## 常见陷阱

1. **为了结构而结构**：内容很简单时，不要硬套复杂结构
2. **结构太复杂**：层级太多，反而找不到重点
3. **只有结构，没有内容**：框架很漂亮，但里面空空洞洞
4. **结论藏在最后**：把最核心的结论放在最后面，读者要找半天
5. **过度包装**：用太多格式、图标、颜色，反而喧宾夺主

## 与其他工具的配合

- **feynman-technique**：费曼技巧保证内容简单易懂，结构化输出保证形式清晰有条理
- **chain-of-thought**：链式思考提供详细推理过程，结构化输出把推理过程组织成清晰的"核心结论 + 推理链"
- **cognitive-tool-orchestrator**：所有工具的最终输出，都可以用结构化输出整理后再交付
